Intro

DBMS or Database Management system is a collection of interrrelated data and a set of programs to access these data. Collection of data is called Database.

Purpose

  • Data redundancy and inconsistency : Duplicates / Unmatching duplicates
  • Difficulty in accessing data
  • Data Isolation: Scattered data in various formats
  • Integrity Problem: Data should follow consistency constraints.
  • Atomicity Problem: Transaction should happen entirely or not at all.
  • Concurrent Access Anomalies
  • Security Problems: Who has access to what part of data

View of Data - Database System provides users with abstract view of the data. Hides storage and maintenance details.

Data Models:

  • Relational Model: Uses tables to represent data and relation. record based.
  • Entitiy-Relation model: Collection of objects called entities and their relationships
  • Semi structured data model
  • Object Based Data Model

Data Abstraction: Physical Level is a low level complex data structures where data is stored. Logical Level is a higher level of abstraction showing what data are stored and what relationships exist between them. (Physical data independence). View Level describes a part of the database

Collection of information stored in a database at a particular moment is called an Instance

Overall Design of database is called Schema. Physical and Logical schemas for Physical and Logical levels respectively.

Database Languages

  • DDL (Data definition language)— Specifying schema
  • DML (Data Manipulation Language)— Express queries and updates

Data definition language : Data storage and definition.

  • Domain Constraints
  • Referential Integrity [?]
  • Authorization — Read, Insert, Update, Delete Authorization

Data Manipulation Language— retrieval, insertion, modification, deletion of data

1. Procedural DML: What data and How to get it

2. Declarative DML: What data

Query is information retrieval request.

Database Engine

1. Storage Manager:

  • Authorization and Integrity Manager
  • Transaction manager
  • File Manager : Allocations and data structures.
  • Buffer Manager : Disk and cache. Important for huge database.

Storage Manager data structures:

  • Data files
  • Data dictionary : Store metadata about database schema
  • Indices: Help accessing data items fast.

2. Query Processor

  • DDL Interpreter
  • DML Compiler
  • Query-evaluation engine: Executes DML

3. Transaction Management

  • Atomicity
  • Consistency
  • Durability

Relational Languages

Structure of relational database

A relational database consists of a collection of tables. A row / tuple in a table represents a relationship among a set of values. Attribute refers to the column of a table. Set of permitted values for an attribute, called the domain of that attribute.
